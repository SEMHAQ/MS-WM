# 讨论

## 精度与速度的权衡

SSM-WM的核心价值在于: 在保持竞争力的预测精度的同时，将推理时间降低至实时控制所需的毫秒级。在合成数据集上SSM-WM精度低于LSTM (MSE差距约167%)，但在推理速度上具有7.3倍的优势。在MuJoCo上SSM-WM精度反而优于LSTM约6%，推理时间9.5ms仍在10ms实时控制预算内。

## 数据集差异分析

合成数据集近线性特性有利于LSTM的非线性递推结构，SSM-WM在该数据集上的精度劣势并不代表真实场景表现。MuJoCo包含真实接触动力学，SSM-WM的精度反而优于LSTM，更能反映SSM在复杂场景下的实际竞争力。消融实验也支持这一分析: 门控机制在MuJoCo上的贡献 (4.6%) 显著大于合成数据集 (2.2%)。

## SSM-WM与Mamba-WM的比较

在MuJoCo上两者精度差距仅1.5%且不显著 (p=0.27)。SSM-WM优势: 实现更简单 (无需自定义CUDA算子)、推理快16%、参数少14%。在对部署简便性和推理速度有严格要求的场景下，SSM-WM是更实用的选择。

## SSM架构选择的依据

首先，SSM的线性递推结构具有O(1)的单步推理延迟，适合在线控制场景。其次，O(T log T)训练复杂度在长序列下优于Transformer的O(T²)。再次，SSM参数量与序列长度无关。最后，SSM在控制理论中有明确的物理意义 (线性系统)，便于与传统控制方法结合。
